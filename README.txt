1. **อธิบายโค้ดที่ป้องกันการ lock เงินไว้ใน contract**
    หากผู้ใช้มีครบ 2 คนและมีผู้ใช้คนใดคนหนึ่ง ทำการเลือก Input ที่จำเล่นแล้วผู้ใช้อีกคนไม่เลือก จนครบ 120 วิ
    จะทำการนำผู้เล่นทั้ง 2 คนออกด้วยฟังก์ชัน checkTimeoutTwoPlayers และคืนเงิน 1 ETH ให้กับผู้เล่นแต่ละคน
2. **อธิบายโค้ดส่วนที่ทำการซ่อน choice และ commit**
    เมื่อเราเลือก Choice แล้วเราก็เอาค่าที่เลือกไปแก้ไขใน choice_hiding_v2 เพื่อให้มันสร้างการสุ่มตัวเลขขนาด 248 บิต 
    และนำค่าที่เลือกรวมเข้ากับค่าที่สุ่มมา เพื่อให้เป็นข้อมูลขนาด 32 ไบต์ จากนั้นนำค่าที่ได้ นำไปใช้กับฟังก์ชัน getHash 
    ในไฟล์ convert.sol เพื่อใช้สำหรับแฮชข้อมูลที่สร้างขึ้นจากโค้ด Python โดยคำนวณ Keccak-256 hash และคืนผลลัพธ์
3. **อธิบายโค้ดส่วนที่จัดการกับความล่าช้าที่ผู้เล่นไม่ครบทั้งสองคนเสียที**
    ฟังก์ชัน checkTimeout หากผู้เล่นเข้ามาก่อน 1 คนและไม่ครบ 2 คน ภายในเวลา 120 วินาที 
    จะให้ผู้เล่นที่เข้ามารอก่อนเด้งออก และคืน ETH ให้กับผู้เล่น และ Reset ค่าในเกม _resetGame
